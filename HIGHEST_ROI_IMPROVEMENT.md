# Highest ROI Improvement

## The One Change

Move the attack action into the first viewport as a large, explicit, game-worded call to action immediately under the first boss image:

**Attack This Boss**

Supporting microcopy:

**Takes 10 seconds. Opens a GitHub issue form. The bot rolls damage, drops loot, comments your result, and closes the issue.**

## Why This Is The Highest ROI Change

The product's biggest problem is not lack of systems. It already has bosses, phases, loot, leaderboards, inventories, Legendary/Mythic history, and SVG identity.

The biggest problem is conversion friction.

Most visitors will not scroll. Most visitors will not infer that opening an issue is welcome. Most visitors will not read far enough to understand loot and status. The current action link is present, but it is visually and emotionally weak.

Changing the first-viewport call to action is the highest ROI because it directly targets:

- Participation: makes the desired action obvious immediately.
- Return visits: more first attacks create more inventory, leaderboard, and boss progress hooks.
- Sharing: more participants produce more public proof, rare drops, and social artifacts.

No other single improvement is as close to the bottleneck.

## Why Not More Bosses?

More bosses improve depth, not conversion. A visitor who does not attack the first boss will not care that there are six bosses.

## Why Not More Loot?

More loot improves collection depth, not first action. A visitor who does not understand the attack loop will never reach the loot loop.

## Why Not More Visuals?

The visuals are already strong enough to create curiosity. The missing step is turning curiosity into action.

## Why Not More Mechanics?

More mechanics would increase cognitive load. The current system already risks feeling complicated. The highest ROI move is to simplify the first decision, not add more decisions.

## Expected Impact

This one change would likely:

- Increase first-time attacks from curious developers.
- Reduce hesitation around issue-form participation.
- Make the game feel intentionally interactive rather than passively decorative.
- Improve recruiter/developer comprehension in under 15 seconds.

## Current Bottleneck

The current README says:

`Open an attack issue`

That sounds like GitHub administration.

The README should say:

`Attack This Boss`

That sounds like play.

## Brutal Summary

The project does not need more features right now. It needs a clearer invitation.

The fastest path to more participation is making the first click feel obvious, safe, and fun.
